49 ## Funktionsbeschreibung
50 
51 If Teams -> Create clicked:
52 - Show *Edit/Create Window*
53 - Hide *Team: Manchester United* so only name of the team textbox available.
54 - After *Save* clicked, show *Unsigned Players List Window* to assign players to the new team.
55 - Show *Edit/Create Window* again.
56 - Hide *Name: Fred Erentz* so we can choose only available teams.
57 - Update *Main Window*.
58 
59 If Teams -> Edit clicked:
60 - Show *Edit/Create Window*
61 - Hide *Team: Manchester United* so only name of the team textbox available.
62 - Update *Main Window*.
63 
64 If Players -> Edit clicked:
65 - Show *Edit/Create Window*+ as is.
66 - Update *Main Window*.
67 
68 If Players -> Create clicked:
69 - Show *Edit/Create Window* as is.
70 - Update *Main Window*.
71 
72 If Unsigned Players clicked:
73 - Show *Unsigned Players Window*.
74  If *Add To Team* clicked:
75  - Show *Edit/Create Window*.
76  - Disable *Name: Fred Erentz* text box from editing.
77  - Update *Main Window*.

81 ### relationale Datenbank: MSSQL

85 ### nicht-relationale Datenbank: MongoDB
86 
87 #### local mongoDB server:
88 
89 - download from [https://www.mongodb.com/](https://www.mongodb.com/) and install
90 - create the folder:
91  ```shell
92  C:\data\db\
93  ```
94 - add install directory bin folder to system path
95 - run server in cmd.exe with:
96  ```shell
97  mongod
98  ```
99  (and leave it running in the background)
100 - to connect to server with mongoshell in cmd.exe use:
101  ```shell
102  mongo
103  ```
104 - to use local DB with project switch from *mlab connection* in DB.cs to commented out *local connection* code
105 
106 #### hosted mongoDB server on mLab:
107 
108 - reachable through mongo shell:
109  ````shell
110  mongo ds241065.mlab.com:41065/as-three-tier-architecture -u <dbuser> -p <dbpassword>
111  ```
112 - reachable in c# through mongo driver connection string:
113  ```shell
114  mongodb://<dbuser>:<dbpassword>@ds241065.mlab.com:41065/as-three-tier-architecture
115  ```
116  set system environment variables with .env.bat, *which has been sent out to team via mail, just rename and run the file or set the values included manually*)

118 #### Tables / Collections
119 
120 To avoid mutable, growing arrays in team, we store the team reference inside the player document:
121 
122 **team collection:**
123 
124 ```json
125 {
126  "_id": "1b9db300-7f60-439e-814c-371b25155dd1",
127  "name": "Team One"
128 }
129 ```
130 
131 **player collection:**
132 
133 ```json
134 {
135  "_id": "2e2c46f3-a588-4e80-aa48-074331c39242",
136  "Name": "Player One",
137  "Team": "1b9db300-7f60-439e-814c-371b25155dd1"
138 }
139 
140 {
141  "_id": "24bea8f8-9d4c-4eac-b808-1471f558122c",
142  "Name": "Player Two",
143  "Team": "1b9db300-7f60-439e-814c-371b25155dd1"
144 }
145 ```
